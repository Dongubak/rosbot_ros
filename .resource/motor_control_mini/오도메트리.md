코드 구조
통신 흐름

Timer (30Hz)
  └─→ HTTP GET /js?json={"T":130}
        └─→ 응답: {"odl": <left_tick>, "odr": <right_tick>, ...}
              └─→ 차동 구동 운동학 계산
                    └─→ nav_msgs/Odometry → /odometry/wheels 발행
                    └─→ TF: odom → base_link
운동학 계산

dl = delta_odl * ticks_to_meter   # 왼쪽 이동거리
dr = delta_odr * ticks_to_meter   # 오른쪽 이동거리

d_center = (dl + dr) / 2          # 직진 거리
d_theta  = (dr - dl) / 0.153      # 회전각 (wheel_separation)

x += d_center * cos(theta + d_theta/2)
y += d_center * sin(theta + d_theta/2)
theta += d_theta

# ROS2 환경 설정
source /opt/ros/humble/setup.bash

# 파라미터 override 예시
python3 .resource/motor_control_mini/cobra_flex_odom_node.py \
  --ros-args \
  -p controller_ip:=192.168.0.44 \
  -p ticks_to_meter:=0.001 \
  -p publish_tf:=false
필수 교정: ticks_to_meter
지금 기본값 0.001 (1 tick = 1mm)은 추정값입니다. 실제 로봇에서 확인 필요:


# 터미널 1: 노드 실행 후 raw 응답 확인용 (임시)
# 로봇을 정지 상태에서 실행 → 초기값 확인

# 터미널 2: 토픽 모니터링
ros2 topic echo /odometry/wheels

# 교정 절차:
# 1. 노드 실행 → 초기 odl, odr 로그 기록
# 2. 로봇을 정확히 1m 직진
# 3. odl 변화량 확인 (예: 1000 변화 → ticks_to_meter = 1.0/1000 = 0.001)
