# 01. 프로젝트 개요

> [!NOTE] 이 문서의 목적
> rosbot_ros 레포지토리가 무엇인지, 어떤 로봇을 지원하는지, 전체적인 맥락을 이해한다.

---

## 이 프로젝트는 무엇인가?

**rosbot_ros**는 Husarion 사가 만든 ROSbot 시리즈 모바일 로봇을 위한 **ROS 2 소프트웨어 스택**이다.

> [!TIP] 비유로 이해하기
> 레포지토리 전체를 "자동차의 전자 제어 시스템"에 비유할 수 있다.
> - 엔진(모터·센서) ↔ **rosbot_hardware_interfaces** (하드웨어 통신)
> - 속도계·계기판 ↔ **ros2_control + 컨트롤러** (상태 관리·명령 전달)
> - 내비게이션 시스템 ↔ **rosbot_localization** (위치 추정)
> - 공장 출하 사양서 ↔ **rosbot_description** (로봇 형태 정의 URDF)
> - 시동 버튼 ↔ **rosbot_bringup** (launch 파일)

---

## 지원하는 로봇 모델

| 모델 | 구동 방식 | 기본 구동 컨트롤러 | 특징 |
|------|-----------|--------------------|------|
| **ROSbot** | 차동 구동 (Differential) | diff_drive_controller | 일반 4륜, 소형 |
| **ROSbot XL** | 메카넘 휠 (Mecanum, 기본) | mecanum_drive_controller | 대형, 옴니 가능, 매니퓰레이터 장착 가능 |

> [!WARNING] Cobra Flex와의 관계
> 우리는 ROSbot의 코드를 기반으로 **Cobra Flex** 로봇에 맞게 수정한다.
> **ROSbot XL 관련 코드(메카넘, 매니퓰레이터)는 우리와 무관하다.**

---

## 지원하는 환경

레포지토리는 두 가지 실행 환경을 지원한다.

```
HUSARION_ROS_BUILD_TYPE=hardware     → 실제 로봇 구동
HUSARION_ROS_BUILD_TYPE=simulation   → Gazebo 시뮬레이션
```

> [!TIP] 비유로 이해하기
> "hardware"는 실제 자동차를 운전하는 것이고,
> "simulation"은 레이싱 게임으로 먼저 연습하는 것이다.

---

## 지원하는 Configuration(구성)

`configuration` 인자로 로봇의 용도를 선택한다.

| Configuration | 설명 | 우리에게 해당? |
|---------------|------|---------------|
| `basic` | LiDAR + 카메라 기본 구성 | **✅ 참고 대상** |
| `telepresence` | 원격 영상 조종 | ❌ |
| `autonomy` | 자율 주행 | ❌ |
| `manipulation` | 매니퓰레이터 포함 | ❌ |
| `manipulation_pro` | 고급 매니퓰레이터 | ❌ |
| `custom` | 사용자 정의 | ❌ |

## 핵심 의존 패키지 (외부)

이 레포지토리 자체가 구현하지 않고 의존하는 중요한 외부 패키지들:

| 패키지 | 역할 |
|--------|------|
| `ros2_control` | 하드웨어 추상화 프레임워크 |
| `diff_drive_controller` | 차동 구동 컨트롤러 |
| `robot_localization` | EKF 센서 융합 |
| `laser_filters` | 레이저 스캔 필터링 |
| `micro_ros_agent` | MCU↔ROS2 통신 |
| `husarion_components_description` | 센서 URDF 컴포넌트 |
| `teleop_twist_joy` | 조이스틱 → Twist 변환 |

> [!NOTE] 중요
> `husarion_components_description`은 ROSbot 전용 센서(LDR02, LDR06, CAM01)를 정의한다.
> **Cobra Flex에서는 이 패키지 대신 RPLidar A2M1과 Gemini336에 맞는 URDF를 직접 작성해야 한다.**
