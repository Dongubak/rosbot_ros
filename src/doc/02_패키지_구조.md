# 02. 패키지 구조

> [!NOTE] 이 문서의 목적
> 9개 패키지 각각이 무슨 역할을 하는지 이해한다.

---

## 전체 의존 관계

```
rosbot (메타패키지)
├── rosbot_bringup         ← 실제 로봇 시작점
│   ├── rosbot_controller  ← 구동 컨트롤러 관리
│   ├── rosbot_joy         ← 조이스틱 조종
│   ├── rosbot_localization← EKF 위치 추정
│   └── rosbot_utils       ← 레이저 필터 등 유틸
├── rosbot_description     ← 로봇 URDF 모델 (공통)
├── rosbot_hardware_interfaces ← C++ 하드웨어 드라이버
└── rosbot_gazebo          ← 시뮬레이션 시작점
    └── (위와 동일한 하위 패키지들)
```

> [!TIP] 비유로 이해하기
> - `rosbot_bringup` = 사장님 (모든 것을 지시)
> - `rosbot_controller` = 운전기사 (모터 제어)
> - `rosbot_description` = 설계도 (로봇의 형태)
> - `rosbot_hardware_interfaces` = 기계실 기술자 (하드웨어와 직접 대화)
> - `rosbot_localization` = 내비게이션 (내가 어디 있는지 파악)
> - `rosbot_utils` = 정비팀 (필터링, 펌웨어 업데이트)
> - `rosbot_joy` = 리모컨 담당
> - `rosbot_gazebo` = 시뮬레이션 담당 사장님

---

## 패키지별 상세 설명

### 1. `rosbot` — 메타패키지

```
rosbot/
└── package.xml   ← 의존성 목록만 있음 (코드 없음)
```

**역할**: 다른 패키지들을 하나로 묶어주는 껍데기 패키지.
빌드 타입(`hardware` / `simulation`)에 따라 의존 패키지가 달라진다.

```xml
<!-- hardware일 때 → rosbot_bringup 포함 -->
<!-- simulation일 때 → rosbot_gazebo 포함 -->
```

---

### 2. `rosbot_bringup` — 실제 로봇 시작점

```
rosbot_bringup/
├── launch/
│   ├── bringup.launch.py    ← 메인 진입점 🔑
│   └── microros.launch.py   ← Micro-ROS 에이전트 실행
└── package.xml
```

**역할**: 실제 하드웨어 로봇을 켤 때 사용하는 최상위 launch 파일.
아래 순서로 모든 노드를 실행한다:

1. **rosbot_controller** (컨트롤러 + URDF 로드)
2. **microros.launch.py** (MCU 통신)
3. **ekf.launch.py** (위치 추정)
4. **laser_filter.launch.py** (스캔 필터)
5. **joy.launch.py** (조이스틱)

> [!NOTE]
> `microros.launch.py`는 ROSbot의 STM32와 통신하기 위한 것이다.
> Cobra Flex의 MCU 환경에 따라 수정이 필요하다. → [[10_Cobra_Flex_적용_가이드]]

---

### 3. `rosbot_controller` — 컨트롤러 관리

```
rosbot_controller/
├── launch/
│   ├── controller.launch.py     ← 메인 컨트롤러 launch 🔑
│   └── manipulator.launch.py    ← ❌ 미사용 (매니퓰레이터)
└── config/
    ├── rosbot/
    │   └── controllers.yaml     ← ROSbot 차동 구동 설정 🔑
    └── rosbot_xl/
        └── controllers.yaml     ← ❌ 미사용 (ROSbot XL)
```

**역할**: `ros2_control` 컨트롤러 매니저를 실행하고, 컨트롤러들을 순서대로 띄운다.

실행 순서 (지연 포함):
```
t=0s   : Controller Manager 시작 + URDF 로드
t=0s   : diff_drive_controller 스폰
t=0s   : imu_sensor_broadcaster 스폰
t=2s   : joint_state_broadcaster 스폰
t=5s   : (manipulator 컨트롤러 - 우리 미사용)
```

> [!TIP]
> `controllers.yaml`의 차동 구동 파라미터(휠 간격, 반지름 등)는
> **Cobra Flex 치수에 맞게 수정해야 한다.** → [[10_Cobra_Flex_적용_가이드]]

---

### 4. `rosbot_description` — 로봇 URDF 모델

```
rosbot_description/
├── launch/
│   ├── load_urdf.launch.py   ← URDF 로드 + robot_state_publisher
│   └── rviz.launch.py        ← RViz 시각화
├── config/
│   └── rosbot/
│       └── basic.yaml        ← 센서 컴포넌트 선택 (LDR02, CAM01)
└── urdf/
    ├── rosbot.urdf.xacro     ← ROSbot 최상위 URDF 🔑
    └── rosbot/
        ├── rosbot_macro.urdf.xacro  ← 본체 구조
        ├── body.urdf.xacro          ← 섀시
        ├── wheel.urdf.xacro         ← 바퀴 (diff/mecanum 분기)
        └── components.urdf.xacro    ← 외부 센서 마운트
```

**역할**: 로봇의 3D 형태, 관절, 센서 위치를 정의한다. TF 트리의 기반이 된다.

> [!IMPORTANT]
> ROSbot의 URDF는 `husarion_components_description` 패키지를 통해 센서를 추가한다.
> Cobra Flex의 RPLidar A2M1과 Gemini336 카메라는 이 패키지에 없으므로
> **URDF에 직접 센서 링크를 추가해야 한다.** → [[10_Cobra_Flex_적용_가이드]]

---

### 5. `rosbot_hardware_interfaces` — C++ 하드웨어 드라이버

```
rosbot_hardware_interfaces/
├── src/
│   ├── rosbot_system.cpp       ← 모터 제어 인터페이스 🔑
│   └── rosbot_imu_sensor.cpp   ← IMU 센서 인터페이스 🔑
└── include/
    └── rosbot_hardware_interfaces/
        ├── rosbot_system.hpp
        └── rosbot_imu_sensor.hpp
```

**역할**: `ros2_control`과 실제 하드웨어(MCU) 사이의 번역기.

두 클래스가 있다:
- **`RosbotSystem`**: 4개 바퀴 속도 명령 전송 + 위치/속도 피드백 수신
- **`RosbotImuSensor`**: IMU 데이터 수신

통신 경로:
```
RosbotSystem ←→ /_motors_cmd, /_motors_response (Micro-ROS 토픽)
RosbotImuSensor ←→ ~/imu (Micro-ROS 토픽)
```

> [!NOTE]
> 이 C++ 코드는 Micro-ROS 기반의 ROSbot STM32 펌웨어와 통신하도록 설계되어 있다.
> Cobra Flex의 MCU가 같은 방식을 쓴다면 재사용 가능하고, 다르면 수정이 필요하다.

---

### 6. `rosbot_joy` — 조이스틱 조종

```
rosbot_joy/
├── launch/
│   └── joy.launch.py      ← joy_node + teleop_twist_joy 실행
└── config/
    └── joy.yaml           ← 버튼 매핑, 속도 설정
```

**역할**: USB 조이스틱(게임패드) 입력을 `/cmd_vel` Twist 메시지로 변환.

주요 설정:
```yaml
device: /dev/input/js0
deadzone: 0.1
enable_button: LB (버튼 4)    # 누르고 있어야 이동
turbo_button: RB (버튼 5)     # 빠른 속도
linear_x: axis 1 (scale 0.7 m/s)
angular_z: axis 3 (scale 1.0 rad/s)
```

---

### 7. `rosbot_localization` — EKF 위치 추정

```
rosbot_localization/
├── launch/
│   └── ekf.launch.py    ← robot_localization EKF 노드 실행
└── config/
    └── ekf.yaml         ← 센서 융합 파라미터 🔑
```

**역할**: 바퀴 오도메트리 + IMU 데이터를 합쳐서 더 정확한 위치를 추정한다.

> [!TIP] 비유로 이해하기
> 바퀴 오도메트리 단독 사용 = GPS 없이 이동 거리만으로 위치 추정 (오차 누적)
> IMU 단독 사용 = 자이로만으로 방향 추정 (드리프트 발생)
> **EKF = 두 정보를 통계적으로 최적 결합** → 더 안정적인 위치 추정

입력: `/odometry/wheels` + `/imu/data`
출력: `/odometry/filtered`

---

### 8. `rosbot_utils` — 유틸리티

```
rosbot_utils/
├── launch/
│   └── laser_filter.launch.py  ← 레이저 박스 필터 실행
├── config/
│   └── rosbot/
│       └── laser_filter.yaml   ← 필터 영역 설정 🔑
└── rosbot_utils/
    ├── flash_firmware.py       ← STM32 펌웨어 업로드
    └── create_config_dir.py    ← 설정 디렉토리 생성
```

**역할**: 레이저 스캔에서 로봇 자신의 몸체가 찍히는 부분을 잘라낸다.

```yaml
# ROSbot 기준 필터 영역 (로봇 풋프린트)
x: -0.17 ~ 0.10 m
y: -0.12 ~ 0.12 m
z: 0.0 ~ 0.2 m
```

> [!IMPORTANT]
> 이 필터 영역은 **Cobra Flex 로봇의 실제 크기에 맞게 수정해야 한다.**
> 잘못 설정하면 로봇 자신이 장애물로 인식되거나, 실제 장애물이 무시된다.

---

### 9. `rosbot_gazebo` — Gazebo 시뮬레이션

```
rosbot_gazebo/
├── launch/
│   ├── simulation.launch.py    ← 시뮬레이션 메인 진입점
│   └── spawn_robot.launch.py   ← 추가 로봇 스폰
└── config/
    ├── gz_bridge.yaml          ← Gazebo → ROS2 토픽 브릿지
    └── rosbot_bridge.yaml      ← 로봇별 토픽 브릿지
```

**역할**: Gazebo 시뮬레이터에서 ROSbot을 실행한다. 실제 하드웨어 없이 테스트 가능.

| 모터 드라이버 | `rosbot_hardware_interfaces/src/rosbot_system.cpp` |
| IMU 드라이버 | `rosbot_hardware_interfaces/src/rosbot_imu_sensor.cpp` |
