# 08. 런치 시스템

> [!NOTE] 이 문서의 목적
> `ros2 launch` 명령어 하나로 어떻게 여러 노드가 순서대로 실행되는지,
> 각 launch 파일이 어떤 관계인지 이해한다.

---

## 런치 시스템이란?

> [!TIP] 비유로 이해하기
> launch 파일은 "도미노"와 같다.
> 첫 번째 launch 파일을 실행하면 다른 launch 파일들을 연쇄적으로 불러오고,
> 결국 필요한 모든 노드가 자동으로 실행된다.

---

## 실제 로봇 실행 흐름

```
ros2 launch rosbot_bringup bringup.launch.py
         │
         ├── rosbot_controller/launch/controller.launch.py
         │      ├── rosbot_description/launch/load_urdf.launch.py
         │      │      └── robot_state_publisher 노드
         │      └── ros2_control_node
         │             ├── diff_drive_controller 스폰
         │             ├── imu_sensor_broadcaster 스폰
         │             └── joint_state_broadcaster 스폰
         │
         ├── rosbot_bringup/launch/microros.launch.py
         │      └── micro_ros_agent 노드 (UART/UDP)
         │
         ├── rosbot_localization/launch/ekf.launch.py
         │      └── ekf_node
         │
         ├── rosbot_utils/launch/laser_filter.launch.py
         │      └── laser_filter_node
         │
         └── rosbot_joy/launch/joy.launch.py
                ├── joy_node
                └── teleop_node
```

## 실행 명령어

### 실제 하드웨어

```bash
# 기본 실행 (ROSbot 모델, basic 구성)
ros2 launch rosbot_bringup bringup.launch.py

# 주요 인자들
ros2 launch rosbot_bringup bringup.launch.py \
    robot_model:=rosbot \         # rosbot 또는 rosbot_xl
    namespace:=robot1 \            # 멀티로봇 시 네임스페이스
    serial_port:=/dev/ttySERIAL   # MCU 시리얼 포트
```

### Gazebo 시뮬레이션

```bash
# 기본 시뮬레이션 실행
ros2 launch rosbot_gazebo simulation.launch.py

# 주요 인자들
ros2 launch rosbot_gazebo simulation.launch.py \
    robot_model:=rosbot \
    mecanum:=False \               # 차동 구동 사용
    gz_headless_mode:=True \       # GUI 없이 실행 (서버 환경)
    rviz:=True                     # RViz 함께 실행
```

## 주요 Launch 파라미터

### bringup.launch.py

| 파라미터 | 기본값 | 설명 |
|---------|--------|------|
| `robot_model` | `rosbot` | `rosbot` 또는 `rosbot_xl` |
| `configuration` | `basic` | 센서 구성 (basic, autonomy 등) |
| `mecanum` | `False` | 메카넘 휠 여부 (ROSbot은 항상 False) |
| `namespace` | `""` | 멀티로봇 네임스페이스 |
| `serial_port` | `/dev/ttySERIAL` | Micro-ROS 시리얼 포트 |

### simulation.launch.py

| 파라미터 | 기본값 | 설명 |
|---------|--------|------|
| `robot_model` | `rosbot` | 로봇 모델 |
| `mecanum` | 모델에 따라 | 메카넘 여부 |
| `gz_world` | 기본 세계 | Gazebo 세계 파일 경로 |
| `gz_headless_mode` | `False` | GUI 없이 실행 여부 |
| `gz_log_level` | `1` | 로그 레벨 (0=verbose, 4=error만) |
| `rviz` | `False` | RViz 실행 여부 |
| `x, y, z` | `0.0` | 로봇 초기 위치 |
| `roll, pitch, yaw` | `0.0` | 로봇 초기 자세 |

## controller.launch.py 내부 동작

```python
# 1단계: URDF 로드
IncludeLaunchDescription(load_urdf.launch.py, {
    'robot_model': robot_model,
    'mecanum': mecanum,
    'configuration': configuration,
    'use_sim': use_sim,
    'namespace': namespace
})

# 2단계: ros2_control 노드 시작 (Controller Manager)
Node(
    package='controller_manager',
    executable='ros2_control_node',
    parameters=[robot_description, controllers_config]
)

# 3단계: 컨트롤러 순서대로 스폰
# 즉시
SpawnEntityNode('diff_drive_controller')
SpawnEntityNode('imu_sensor_broadcaster')
# 2초 후
TimerAction(2.0, SpawnEntityNode('joint_state_broadcaster'))
```

## load_urdf.launch.py 내부 동작

```python
# xacro → URDF 변환
# 주요 xacro 인자들:
xacro_args = {
    'mecanum': mecanum,              # 바퀴 타입
    'use_sim': use_sim,              # 시뮬레이션 여부 (하드웨어 인터페이스 분기)
    'configuration': configuration,  # 센서 구성
    'namespace': namespace,
    'camera_model': camera_model,
    'lidar_model': lidar_model,
}

# robot_state_publisher 실행
Node(
    package='robot_state_publisher',
    executable='robot_state_publisher',
    parameters=[{'robot_description': urdf_content}]
)
```

## 환경변수로 빌드 타입 지정

```bash
# 실제 로봇용 빌드
export HUSARION_ROS_BUILD_TYPE=hardware
colcon build

# 시뮬레이션용 빌드
export HUSARION_ROS_BUILD_TYPE=simulation
colcon build
```

이 환경변수에 따라 `rosbot/package.xml`이 다른 패키지를 의존한다.

## 유용한 디버깅 명령어

```bash
# 실행 중인 노드 목록 확인
ros2 node list

# 토픽 목록 확인
ros2 topic list

# 특정 토픽 실시간 확인
ros2 topic echo /odometry/filtered

# TF 트리 확인
ros2 run tf2_tools view_frames

# 컨트롤러 상태 확인
ros2 control list_controllers

# 하드웨어 인터페이스 상태 확인
ros2 control list_hardware_interfaces
```
