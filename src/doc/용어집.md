# 용어집

> 이 문서는 rosbot_ros 코드베이스에서 자주 등장하는 전문 용어를 정리한 참고 자료다.
> 알파벳 → 한글 순으로 정리한다.

---

## A–F

### AMCL (Adaptive Monte Carlo Localization)
**한국어**: 적응형 몬테카를로 위치 추정

이미 만들어진 지도 위에서 로봇의 위치를 추정하는 알고리즘.
확률론적 방법(파티클 필터)을 사용한다.

> 비유: 지도를 들고 "내가 지금 지도의 어느 점에 있는가?"를 추정하는 것.
> 이 레포는 AMCL을 포함하지 않는다. EKF만 포함.

---

### Base Link
로봇 좌표계의 기준 프레임. 보통 로봇 본체 중심 하단에 위치.
모든 다른 프레임(센서, 바퀴)은 base_link를 기준으로 위치가 정의된다.

> 비유: 인체에서 배꼽이 기준점인 것처럼, base_link는 로봇의 배꼽.

---

### Broadcaster (브로드캐스터)
ros2_control에서 센서 데이터를 ROS2 토픽으로 발행하는 컨트롤러.
예: `imu_sensor_broadcaster`, `joint_state_broadcaster`

---

### cmd_vel
로봇에 속도 명령을 내리는 표준 토픽 이름.
메시지 타입: `geometry_msgs/Twist`

```
linear.x  : 앞뒤 속도 (m/s)
linear.y  : 좌우 속도 (m/s, 차동 구동은 항상 0)
angular.z : 회전 속도 (rad/s, 양수=왼쪽 회전)
```

---

### Covariance (공분산)
센서 측정값의 **불확실성**을 나타내는 행렬.
값이 클수록 해당 측정값을 덜 신뢰한다.
EKF가 센서 융합 비율을 결정할 때 사용한다.

> 비유: 저울의 오차 범위. ±1g 오차인 저울과 ±10g 오차인 저울이 있다면
> EKF는 ±1g 저울을 더 많이 참고한다.

---

### DDS (Data Distribution Service)
ROS2의 통신 미들웨어. 토픽 발행/구독의 실제 전송 계층.
ROS1의 roscore 대신 DDS가 분산 통신을 담당.

---

### Differential Drive (차동 구동)
두 바퀴(또는 양쪽 바퀴 쌍)의 속도 차이로 방향을 바꾸는 구동 방식.
앞바퀴 조향이 없이 좌우 바퀴 속도 차만으로 회전한다.

> 비유: 탱크. 왼쪽 무한궤도가 빠르면 오른쪽으로 회전.

---

### EKF (Extended Kalman Filter, 확장 칼만 필터)
여러 센서의 측정값을 최적으로 결합하는 통계적 필터.
각 센서의 불확실성(covariance)을 고려하여 가중 평균적으로 융합.

> 비유: 두 증인의 증언이 다를 때, 각 증인의 신뢰도를 고려하여 진실을 추정하는 것.
> 신뢰도 높은 증인 쪽에 더 비중을 둔다.

사용 패키지: `robot_localization`

---

### Frame ID
ROS2에서 좌표계를 식별하는 문자열 이름.
예: `base_link`, `odom`, `lidar_link`, `map`

---

## G–M

### Gazebo
ROS2와 통합되는 3D 로봇 시뮬레이터.
물리 엔진 포함, 센서(LiDAR, 카메라, IMU) 시뮬레이션 가능.

---

### Hardware Interface (하드웨어 인터페이스)
ros2_control 프레임워크에서 실제 하드웨어와 통신하는 플러그인.
`SystemInterface` (모터 제어) 또는 `SensorInterface` (센서만) 구현.

---

### IMU (Inertial Measurement Unit, 관성 측정 장치)
가속도계(accelerometer) + 자이로스코프(gyroscope) 조합 센서.
- 가속도계: 선형 가속도 측정 (m/s²)
- 자이로스코프: 각속도 측정 (rad/s)

> 비유: 자동차의 에어백 센서(가속도계) + 게임 컨트롤러 기울기 센서(자이로).

---

### Joint (관절)
URDF에서 두 링크를 연결하는 연결부.
- `fixed`: 움직이지 않음 (센서 마운트 등)
- `continuous`: 무한 회전 (바퀴)
- `revolute`: 제한된 회전 범위

---

### Joint State (관절 상태)
각 관절의 현재 위치, 속도, 힘 정보.
토픽: `/joint_states` (타입: `sensor_msgs/JointState`)

---

### Launch File (런치 파일)
여러 노드와 다른 launch 파일들을 한 번에 실행하는 Python 또는 XML 파일.
`ros2 launch` 명령어로 실행.

---

### Link (링크)
URDF에서 로봇의 각 부분(본체, 바퀴, 센서 등)을 나타내는 강체(rigid body).
링크들이 조인트(joint)로 연결되어 로봇 구조를 형성.

---

### Mecanum Wheel (메카넘 휠)
바퀴 표면에 45° 기울어진 롤러가 달린 특수 바퀴.
X/Y 방향 모두 이동 가능한 전방향(Holonomic) 이동을 구현.

> 비유: 마트 카트처럼 어느 방향으로든 밀 수 있는 바퀴.
> **우리 Cobra Flex는 일반 바퀴를 사용하므로 관련 없다.**

---

### Micro-ROS
임베디드 MCU(STM32 등) 위에서 실행되는 경량 ROS2 클라이언트.
`micro_ros_agent`를 통해 PC의 ROS2와 통신.

> 비유: MCU와 ROS2 PC 사이의 번역 전화기.

---

### Multiplier (승수, 보정 승수)
컨트롤러 설정에서 실측 오차를 보정하는 배율값.
예: `wheel_separation_multiplier: 1.36` = 이론값의 1.36배로 계산.

---

## N–R

### Namespace (네임스페이스)
ROS2에서 노드·토픽·서비스 이름 앞에 붙는 prefix.
여러 로봇이 같은 공간에서 동작할 때 이름 충돌 방지.
예: `/robot1/cmd_vel`, `/robot2/cmd_vel`

---

### Node (노드)
ROS2에서 특정 기능을 수행하는 독립적인 프로세스.
예: `ekf_node`, `diff_drive_controller`, `joy_node`

---

### Odometry (오도메트리)
바퀴 회전수로 계산한 이동 거리와 방향 추정.
슬립과 오차가 누적되는 단점이 있어 IMU와 융합하여 보완.

> 비유: 눈 감고 걸음 수만 세어 위치 추정. 미끄러지면 오차 발생.

---

### Odom Frame
ROS2에서 바퀴 오도메트리 기준 좌표계 이름. 보통 `odom`.
로봇 시작점이 원점이며, 주행하면서 base_link의 위치가 갱신됨.

---

### Package (패키지)
ROS2에서 관련 코드·설정·launch 파일들을 묶은 단위.
`package.xml`과 `CMakeLists.txt` 또는 `setup.py`로 정의.

---

### Pluginlib
ROS2에서 플러그인(동적 라이브러리)을 로드하는 프레임워크.
ros2_control이 하드웨어 인터페이스와 컨트롤러를 동적으로 로드할 때 사용.

---

### Process Noise (프로세스 노이즈)
EKF에서 시스템 모델의 불확실성을 나타내는 파라미터.
"내 수식이 현실을 완벽히 반영하지 못한다"는 것을 표현.

---

### Publisher (발행자) / Subscriber (구독자)
- 발행자: 토픽에 메시지를 보내는 쪽
- 구독자: 토픽에서 메시지를 받는 쪽

> 비유: 유튜브 채널(발행자)과 구독자.

---

### Quaternion (쿼터니언)
3D 공간에서 회전을 나타내는 4개 숫자 (x, y, z, w).
오일러각(roll, pitch, yaw)보다 계산이 안정적이어서 ROS에서 표준으로 사용.

> 비유: x, y, z, w는 복잡해 보이지만 roll/pitch/yaw로 변환 가능.
> w = cos(θ/2), [x,y,z] = sin(θ/2) × 회전축

---

### RCLPY / RCLCPP
ROS2 클라이언트 라이브러리.
- rclpy: Python용
- rclcpp: C++용

---

### Remapping (리매핑)
노드의 기본 토픽/서비스 이름을 실행 시간에 다른 이름으로 바꾸는 것.

```python
# 예시: diff_drive_controller/cmd_vel_unstamped → cmd_vel 으로 리매핑
remappings=[
    ('diff_drive_controller/cmd_vel_unstamped', 'cmd_vel')
]
```

---

### ros2_control
ROS2의 표준 하드웨어 제어 프레임워크.
하드웨어 인터페이스(Hardware Interface) + 컨트롤러(Controller) 계층 구조.

> 비유: 전자기기의 표준 USB 소켓.
> 어떤 기기든 USB 규격만 맞으면 같은 케이블로 연결 가능한 것처럼,
> ros2_control 규격을 구현하면 어떤 로봇에도 같은 컨트롤러 사용 가능.

---

## S–Z

### SLAM (Simultaneous Localization and Mapping)
지도를 만들면서 동시에 자신의 위치를 추정하는 기술.
이 레포는 SLAM을 포함하지 않음 (EKF까지만).

> 비유: 낯선 건물에서 지도를 그리면서 동시에 내 위치를 파악하는 것.

---

### Spawn (스폰)
ros2_control에서 컨트롤러를 실행(활성화)하는 것.
`spawner` 도구를 사용하여 Controller Manager에 컨트롤러를 등록하고 활성화.

---

### STM32
ST Microelectronics의 ARM Cortex-M 기반 마이크로컨트롤러.
ROSbot의 모터 제어와 센서 읽기에 사용됨. Micro-ROS로 ROS2와 통신.

---

### TF (Transform)
ROS2에서 좌표계 간의 변환 관계를 관리하는 시스템.
`/tf`와 `/tf_static` 토픽으로 발행.

- `/tf_static`: 변하지 않는 변환 (센서 → 본체 등 고정 관계)
- `/tf`: 시간에 따라 변하는 변환 (odom → base_link 등)

> 비유: GPS 좌표계와 지도 좌표계 변환표.
> "여기서 북쪽으로 10m 가면 어디야?"를 계산하는 시스템.

---

### Topic (토픽)
ROS2에서 노드 간 데이터 교환에 사용되는 명명된 채널.
발행자(publisher)가 메시지를 보내면 구독자(subscriber)가 받는다.

---

### UART (Universal Asynchronous Receiver-Transmitter)
직렬 통신 프로토콜. MCU와 PC 간 시리얼 통신에 사용.
ROSbot에서 STM32와 Micro-ROS 통신에 사용 (576000 baud).

---

### URDF (Unified Robot Description Format)
로봇의 3D 구조(링크, 관절, 센서 위치)를 XML 형식으로 정의하는 표준.
파생 형식: xacro (URDF + 매크로 기능)

---

### Xacro
URDF에 매크로(함수), 변수, 조건문 기능을 추가한 확장 형식.
`*.urdf.xacro` 확장자 사용.

```xml
<!-- xacro 변수 예시 -->
<xacro:property name="wheel_radius" value="0.0425"/>
<xacro:property name="wheel_mass" value="0.074"/>

<!-- 조건문 예시 -->
<xacro:if value="${mecanum}">
  <!-- 메카넘 휠 정의 -->
</xacro:if>
```

## 약어 정리

| 약어 | 풀네임 | 한국어 |
|------|--------|--------|
| EKF | Extended Kalman Filter | 확장 칼만 필터 |
| IMU | Inertial Measurement Unit | 관성 측정 장치 |
| MCU | Microcontroller Unit | 마이크로컨트롤러 |
| TF | Transform | 좌표 변환 |
| URDF | Unified Robot Description Format | 로봇 설명 형식 |
| SLAM | Simultaneous Localization and Mapping | 동시 위치 추정 및 지도 생성 |
| AMCL | Adaptive Monte Carlo Localization | 적응형 몬테카를로 위치 추정 |
| DDS | Data Distribution Service | 데이터 분산 서비스 |
| ROS | Robot Operating System | 로봇 운영 체제 |
| UART | Universal Asynchronous Receiver-Transmitter | 범용 비동기 송수신기 |
| UDP | User Datagram Protocol | 비연결형 네트워크 프로토콜 |
| ToF | Time of Flight | 비행 시간 (거리 측정 방식) |
