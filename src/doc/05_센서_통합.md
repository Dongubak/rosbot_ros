# 05. 센서 통합

> [!NOTE] 이 문서의 목적
> ROSbot에서 센서가 어떻게 URDF에 정의되고 ROS2 토픽으로 나오는지 이해한다.
> Cobra Flex의 RPLidar A2M1과 Gemini336에 적용할 때 참고한다.

---

## ROSbot 센서 구성

| 센서 | 모델 | 토픽 | 우리 사용 여부 |
|------|------|------|--------------|
| LiDAR | LDR02 (ROSbot 기본) | `/scan` | ⚠️ **RPLidar A2M1으로 교체** |
| 카메라 | CAM01 | `/camera/...` | ⚠️ **Gemini336으로 교체** |
| IMU | BNO055 | `/imu/data` | ✅ 참고 (다른 IMU면 설정 변경) |
| 거리 센서 (ToF) | VL53LOX × 4 | `/range_0~3` | ❌ 미사용 |

---

## 센서 통합 방식

ROSbot은 `husarion_components_description` 패키지를 통해 센서를 URDF에 추가한다.

```
rosbot_description/config/rosbot/basic.yaml
  → 센서 이름 목록 (LDR02, CAM01)
  → rosbot_description/launch/load_urdf.launch.py에서 읽음
  → husarion_components_description 패키지의 xacro 파일 참조
  → 로봇 URDF에 센서 링크·조인트 자동 추가
```

> [!NOTE] Cobra Flex에서는
> `husarion_components_description`에 RPLidar A2M1과 Gemini336이 없다.
> 따라서 **센서 URDF를 직접 작성하거나 제조사 패키지를 사용**해야 한다.
> → [[10_Cobra_Flex_적용_가이드]]

## LiDAR

### ROSbot의 LDR02

ROSbot basic 구성에서 LDR02 (2D LiDAR) 사용:

```
토픽: /scan  (sensor_msgs/LaserScan)
360° 스캔, 수평면
```

### 레이저 필터 (laser_filter)

LiDAR 스캔에는 로봇 자신의 몸체가 찍힌다. 이를 제거하는 박스 필터:

```yaml
# rosbot_utils/config/rosbot/laser_filter.yaml
laser_scan_filters:
  - name: LaserScanBoxFilter
    params:
      max_x: 0.10   # 앞쪽 경계 (m)
      min_x: -0.17  # 뒤쪽 경계 (m)
      max_y: 0.12   # 오른쪽 경계 (m)
      min_y: -0.12  # 왼쪽 경계 (m)
      max_z: 0.20   # 위쪽 경계 (m)
      min_z: 0.0    # 아래쪽 경계 (m)
```

```
입력: /scan (원시 스캔)
출력: /scan_filtered (필터링된 스캔)
```

> [!TIP] 비유로 이해하기
> 자신의 코가 카메라에 찍히는 것처럼, LiDAR도 자신의 몸을 감지한다.
> 박스 필터는 "이 영역 안에 있는 점은 내 몸이니까 무시해"라고 알려주는 것이다.

> [!IMPORTANT] Cobra Flex 수정 필요
> 박스 필터 영역을 **Cobra Flex 로봇의 풋프린트(크기)**에 맞게 변경해야 한다.
> RPLidar A2M1의 장착 위치(높이, 오프셋)도 고려해야 한다.

### RPLidar A2M1 연결 방법 (참고)

ROSbot처럼 `husarion_components_description`을 사용하지 않고
`rplidar_ros` 패키지를 직접 사용하는 방법:

```python
# launch 파일에서 rplidar_ros 직접 실행
Node(
    package='rplidar_ros',
    executable='rplidar_node',
    parameters=[{
        'serial_port': '/dev/ttyUSB0',
        'serial_baudrate': 115200,
        'frame_id': 'lidar_link',
        'angle_compensate': True,
        'scan_mode': 'Standard'
    }]
)
```

## 카메라

### ROSbot의 CAM01

```
네임스페이스: /camera/
주요 토픽:
  /camera/color/image_raw
  /camera/depth/image_raw
  /camera/color/camera_info
```

### Gemini336 (Orbbec)

Gemini336은 Orbbec사의 RGB-D 카메라다. `orbbec_camera` ROS2 패키지를 사용한다.

> [!NOTE]
> Gemini336 드라이버는 별도 패키지를 설치해야 한다.
> URDF에 카메라 링크를 추가하고 launch 파일에서 카메라 드라이버 노드를 실행해야 한다.

## IMU (관성 측정 장치)

### ROSbot의 BNO055

**센서**: Bosch BNO055 (6-DOF: 3축 자이로 + 3축 가속도계)
**연결**: STM32 MCU를 통해 Micro-ROS로 전달

```
토픽:
  /_imu/data_raw  → (rosbot_hardware_interfaces 내부 경유)
  /imu/data       → (imu_sensor_broadcaster 출력)

데이터:
  orientation (쿼터니언 x,y,z,w)
  angular_velocity (rad/s)
  linear_acceleration (m/s²)
```

### 공분산 값 (controllers.yaml)

```yaml
orientation_stddev: 0.0019   # BNO055 데이터시트 기준
angular_velocity_stddev: 0.002  # 실험 측정값
linear_acceleration_stddev: 0.064  # 실험 측정값
```

> [!TIP] 공분산이란?
> "이 센서 값을 얼마나 믿을 수 있는가?"를 나타내는 수치다.
> 값이 클수록 불확실성이 높다 = 덜 믿는다.
> EKF는 이 값을 참고하여 센서 융합 비율을 결정한다.

## ToF 거리 센서 (VL53LOX) — 미사용

> [!WARNING] 미사용
> ROSbot에는 4개의 VL53LOX 거리 센서가 있으나, **Cobra Flex 적용 시 사용하지 않는다.**
> 이 센서의 URDF 링크 (`fl_range`, `fr_range`, `rl_range`, `rr_range`)는
> ROSbot URDF에 포함되어 있지만, Cobra Flex URDF 작성 시 포함하지 않는다.

## Gazebo 시뮬레이션에서 센서

Gazebo 시뮬레이션에서는 `rosbot_description/urdf/common/gazebo.urdf.xacro`가
실제 센서 대신 Gazebo 플러그인으로 센서를 시뮬레이션한다.

```xml
<!-- LiDAR 시뮬레이션 -->
<gazebo reference="lidar_link">
  <sensor type="gpu_lidar">
    <ray>
      <scan>
        <horizontal>
          <samples>360</samples>
          <resolution>1</resolution>
          <min_angle>-3.14159</min_angle>
          <max_angle>3.14159</max_angle>
        </horizontal>
      </scan>
      <range>
        <min>0.15</min>
        <max>12.0</max>
      </range>
    </ray>
  </sensor>
</gazebo>
```

Gazebo 토픽 → ROS2 토픽 변환은 `ros_gz_bridge`가 담당한다.

## 센서 프레임 ID

| 센서 | 프레임 ID | 설명 |
|------|----------|------|
| LiDAR | `lidar_link` | LiDAR 센서 원점 |
| 카메라 | `camera_link` | 카메라 광학 원점 |
| IMU | `imu_link` | IMU 센서 위치 |
| ToF 0~3 | `fl_range` 등 | 거리 센서 위치 (미사용) |

> [!IMPORTANT]
> URDF에서 정의한 프레임 ID와 센서 드라이버에서 설정한 `frame_id`가 **반드시 일치**해야 한다.
> 불일치 시 TF 트리가 깨지고 센서 데이터가 올바른 좌표계로 변환되지 않는다.
