# 04. 하드웨어 인터페이스

> [!NOTE] 이 문서의 목적
> ROS2가 실제 모터·IMU 하드웨어와 어떻게 통신하는지 이해한다.
> `rosbot_hardware_interfaces` 패키지와 `micro_ros_agent`의 역할을 파악한다.

---

## 전체 통신 구조

> [!TIP] 비유로 이해하기
> 이 구조는 "본사 ↔ 공장 통신 시스템"과 같다.
> - **ROS2** = 본사 (고수준 명령 처리)
> - **ros2_control** = 중간 관리자 (표준화된 인터페이스 제공)
> - **rosbot_hardware_interfaces** = 현장 통역사 (본사 언어 ↔ 공장 언어 번역)
> - **micro_ros_agent** = 통신 전용선 (데이터 전달 경로)
> - **MCU(STM32)** = 공장 (실제 모터·센서 제어)

```
ROS2 컨트롤러 (diff_drive_controller)
        ↕  [ros2_control 표준 인터페이스]
rosbot_hardware_interfaces
  ├── RosbotSystem       ←→  /_motors_cmd
  │                          /_motors_response
  └── RosbotImuSensor    ←→  ~/imu
        ↕  [ROS2 토픽]
micro_ros_agent
        ↕  [UART (ROSbot) 또는 UDP (ROSbot XL)]
MCU (STM32)
  ├── 모터 드라이버
  └── IMU (BNO055)
```

## ros2_control 이란?

> [!TIP] ros2_control 비유
> ros2_control은 **"표준화된 플러그 소켓"** 같은 프레임워크다.
> 어떤 하드웨어든 `SystemInterface` 또는 `SensorInterface`를 구현하면
> 동일한 컨트롤러(diff_drive 등)를 그대로 사용할 수 있다.
>
> 가전제품 플러그가 110V든 220V든 어댑터 하나로 해결하는 것과 같다.

ros2_control은 두 계층으로 나뉜다:

| 계층 | 구성요소 | 역할 |
|------|---------|------|
| 상위 | Controller (diff_drive, imu_broadcaster 등) | 고수준 알고리즘 |
| 하위 | Hardware Interface (RosbotSystem 등) | 실제 하드웨어 통신 |

컨트롤러는 하드웨어를 몰라도 된다. `Command Interface`와 `State Interface`만 알면 된다.

## RosbotSystem — 모터 제어

**파일**: `rosbot_hardware_interfaces/src/rosbot_system.cpp`
**플러그인 타입**: `hardware_interface::SystemInterface`

### 관리하는 관절

| 관절 이름 | 위치 |
|-----------|------|
| `fl_wheel_joint` | 앞 왼쪽 |
| `fr_wheel_joint` | 앞 오른쪽 |
| `rl_wheel_joint` | 뒤 왼쪽 |
| `rr_wheel_joint` | 뒤 오른쪽 |

### 인터페이스

```
Command Interface (컨트롤러 → 하드웨어):
  각 바퀴 velocity (rad/s)

State Interface (하드웨어 → 컨트롤러):
  각 바퀴 position (rad)
  각 바퀴 velocity (rad/s)
```

### 실제 동작 순서

```
1. read()  호출 → /_motors_response 토픽에서 바퀴 피드백 읽기
2. 컨트롤러 업데이트 (내부 계산)
3. write() 호출 → /_motors_cmd 토픽으로 속도 명령 전송
```

명령 전송 형식 (`Float32MultiArray`):
```
[rr_velocity, rl_velocity, fr_velocity, fl_velocity]  (순서 주의!)
```

### 연결 상태 감시

```yaml
connection_timeout_ms: 30000   # 30초 안에 응답 없으면 오류
connection_check_period_ms: 1000  # 1초마다 연결 확인
```

## RosbotImuSensor — IMU 센서

**파일**: `rosbot_hardware_interfaces/src/rosbot_imu_sensor.cpp`
**플러그인 타입**: `hardware_interface::SensorInterface`

### 제공하는 State Interface (10개)

```
orientation.x, orientation.y, orientation.z, orientation.w  (쿼터니언)
angular_velocity.x, angular_velocity.y, angular_velocity.z
linear_acceleration.x, linear_acceleration.y, linear_acceleration.z
```

### 동작 방식

IMU 데이터는 비동기적으로 도착하므로 별도 스레드를 사용한다:

```
[별도 스레드: MultiThreadedExecutor]
  ~/imu 토픽 구독 → 내부 버퍼 업데이트
                           ↓
[메인 스레드: read() 호출]
  내부 버퍼 → State Interface 업데이트
```

## Micro-ROS Agent

**실행**: `rosbot_bringup/launch/microros.launch.py`
**패키지**: `micro_ros_agent` (외부 패키지)

### 역할

MCU(STM32 펌웨어) 위에서 실행되는 `micro-ROS`와 ROS2 사이를 연결해주는 브릿지.

```
STM32 (micro-ROS 클라이언트)
        ↕ UART 또는 UDP
micro_ros_agent (ROS2 노드)
        ↕ DDS
ROS2 토픽/서비스
```

### 통신 방식

| 로봇 | 통신 방식 | 기본값 |
|------|----------|--------|
| ROSbot | Serial (UART) | `/dev/ttySERIAL`, 576000 baud |
| ROSbot XL | UDP4 | 포트 8888 |

> [!NOTE] Cobra Flex 적용 시
> Cobra Flex의 MCU 통신 방식에 따라 microros.launch.py의 설정을 변경해야 한다.
> 만약 Micro-ROS를 사용하지 않는 경우, 이 launch 파일을 교체해야 한다.
> → [[10_Cobra_Flex_적용_가이드]]

## URDF에서 하드웨어 인터페이스 선언

`rosbot_description/urdf/common/ros2_control.urdf.xacro` 파일에서
하드웨어 인터페이스 플러그인을 URDF에 등록한다.

```xml
<ros2_control name="rosbot_system" type="system">
  <!-- 실제 하드웨어일 때 -->
  <hardware>
    <plugin>rosbot_hardware_interfaces/RosbotSystem</plugin>
  </hardware>

  <!-- 시뮬레이션일 때 -->
  <hardware>
    <plugin>gz_ros2_control/GazeboSimSystem</plugin>
  </hardware>

  <!-- 바퀴 관절 정의 -->
  <joint name="fl_wheel_joint">
    <command_interface name="velocity"/>
    <state_interface name="position"/>
    <state_interface name="velocity"/>
  </joint>
  <!-- ... -->
</ros2_control>

<!-- IMU 별도 정의 -->
<ros2_control name="rosbot_imu_sensor" type="sensor">
  <hardware>
    <plugin>rosbot_hardware_interfaces/RosbotImuSensor</plugin>
  </hardware>
  <sensor name="imu">
    <state_interface name="orientation.x"/>
    <!-- ... 10개 인터페이스 -->
  </sensor>
</ros2_control>
```

## 펌웨어 업데이트

ROSbot의 STM32 펌웨어는 아래 명령으로 업데이트한다:

```bash
ros2 run rosbot_utils flash_firmware
```

> [!NOTE]
> 이 유틸리티는 ROSbot 전용이다. Cobra Flex의 MCU 펌웨어 업데이트 방법은 별도로 확인해야 한다.
