## 차동 구동 컨트롤러 (diff_drive_controller)

> [!TIP] 차동 구동 비유
> 탱크나 휠체어처럼 왼쪽/오른쪽 바퀴 속도 차이로 방향을 바꾸는 방식이다.
> 왼쪽이 더 빠르면 오른쪽으로, 오른쪽이 더 빠르면 왼쪽으로 회전한다.

**설정 파일**: `rosbot_controller/config/rosbot/controllers.yaml`

```yaml
controller_manager:
  ros__parameters:
    update_rate: 10  # 컨트롤러 업데이트 주기 (Hz)

diff_drive_controller:
  ros__parameters:
    # 바퀴 관절 이름
    left_wheel_names: [fl_wheel_joint, rl_wheel_joint]
    right_wheel_names: [fr_wheel_joint, rr_wheel_joint]

    # 로봇 물리적 치수
    wheel_separation: 0.186      # 좌우 바퀴 간격 (m) 🔑 수정 필요
    wheel_radius: 0.0425         # 바퀴 반지름 (m) 🔑 수정 필요

    # 보정 승수
    wheel_separation_multiplier: 1.36  # 실측 보정값
    left_wheel_radius_multiplier: 0.95
    right_wheel_radius_multiplier: 0.95

    # 오도메트리 발행
    publish_rate: 20.0          # 오도메트리 발행 주기 (Hz)
    odom_frame_id: odom
    base_frame_id: base_link
    pose_covariance_diagonal: [0.001, 0.001, 0.001, 0.001, 0.001, 0.01]
    twist_covariance_diagonal: [0.001, 0.001, 0.001, 0.001, 0.001, 0.01]

    # 속도 제한
    linear.x.has_velocity_limits: true
    linear.x.max_velocity: 1.0   # m/s
    linear.x.min_velocity: -1.0  # m/s
    linear.x.has_acceleration_limits: true
    linear.x.max_acceleration: 1.0  # m/s²

    angular.z.has_velocity_limits: true
    angular.z.max_velocity: 3.14  # rad/s (~180°/s)
    angular.z.has_acceleration_limits: true
    angular.z.max_acceleration: 4.0  # rad/s²
```

### 파라미터 설명

| 파라미터 | 설명 | 예시값 | 수정 필요 |
|---------|------|--------|---------|
| `wheel_separation` | 좌우 바퀴 중심 간격 | 0.186 m | **✅ Cobra Flex 치수로** |
| `wheel_radius` | 바퀴 반지름 | 0.0425 m | **✅ Cobra Flex 치수로** |
| `wheel_separation_multiplier` | 실측 오차 보정 | 1.36 | 캘리브레이션 후 조정 |
| `left/right_wheel_radius_multiplier` | 좌우 바퀴 반지름 보정 | 0.95 | 캘리브레이션 후 조정 |
| `publish_rate` | 오도메트리 발행 주기 | 20 Hz | 필요 시 조정 |

> [!IMPORTANT] Cobra Flex 수정 사항
> `wheel_separation`과 `wheel_radius`는 **반드시** Cobra Flex의 실제 치수를 측정하여 입력해야 한다.
> 잘못된 값은 직선 주행 시 휘거나, 회전 각도가 부정확해진다.

### 보정 승수의 의미

실제 로봇은 제조 오차, 슬립 등으로 이론값과 차이가 난다. 보정 승수로 이를 수정한다.

```
실제 wheel_separation = wheel_separation × wheel_separation_multiplier
실제 바퀴 반지름 = wheel_radius × left/right_wheel_radius_multiplier
```

> [!TIP] 캘리브레이션 방법
> 1. 로봇을 정확히 1m 직진시킨다
> 2. `/odometry/wheels`의 이동 거리를 확인한다
> 3. 오차가 있으면 `radius_multiplier`를 조정한다
> 4. 로봇을 정확히 360° 회전시킨다
> 5. `/odometry/wheels`의 회전각을 확인한다
> 6. 오차가 있으면 `separation_multiplier`를 조정한다

## IMU 센서 브로드캐스터

```yaml
imu_sensor_broadcaster:
  ros__parameters:
    sensor_name: imu     # ros2_control에서 정의한 센서 이름
    frame_id: imu_link   # TF 프레임 ID

    # 센서 공분산 (불확실성)
    static_covariance_orientation: [1.9e-3, 0, 0, 0, 1.9e-3, 0, 0, 0, 1.9e-3]
    static_covariance_angular_velocity: [0, 0, 0, 0, 0, 0, 0, 0, 2.0e-3]
    static_covariance_linear_acceleration: [6.4e-2, 0, 0, 0, 2.2e-2, 0, 0, 0, 0]
```

| 파라미터 | 의미 |
|---------|------|
| `static_covariance_orientation` | 자세 추정 불확실성 (BNO055 데이터시트) |
| `static_covariance_angular_velocity` | 각속도 측정 불확실성 |
| `static_covariance_linear_acceleration` | 선가속도 측정 불확실성 |

> [!NOTE]
> 다른 IMU를 사용한다면 해당 센서의 데이터시트에서 공분산 값을 확인해야 한다.
> 공분산이 EKF 성능에 직접 영향을 미친다. → [[07_로컬라이제이션]]

## 조인트 상태 브로드캐스터

```yaml
joint_state_broadcaster:
  ros__parameters: {}  # 기본값 사용 (모든 관절 자동 포함)
```

`/joint_states` 토픽으로 모든 관절의 위치·속도·힘을 발행한다.
`robot_state_publisher`가 이 값을 사용하여 TF를 계산한다.

## 메카넘 드라이브 컨트롤러 — 미사용

> [!WARNING] 미사용
> ROSbot XL의 옴니휠(메카넘 휠) 구동에 사용되는 컨트롤러다.
> 우리 로봇은 차동 구동 방식이므로 **사용하지 않는다.**

메카넘 휠은 45° 기울어진 롤러가 달린 특수 바퀴로, X/Y 방향 모두 이동 가능하다.
우리 Cobra Flex는 일반 4륜 차동 구동이므로 이 컨트롤러는 관련 없다.

## 컨트롤러 스폰 순서

```
t = 0s  : controller_manager 시작
t = 0s  : diff_drive_controller 스폰
t = 0s  : imu_sensor_broadcaster 스폰
t = 2s  : joint_state_broadcaster 스폰 (2초 지연)
```

> [!NOTE] 왜 지연이 있는가?
> `joint_state_broadcaster`는 `diff_drive_controller`가 이미 관절 상태를 등록한 후에 스폰되어야 한다.
> 2초 지연은 이 초기화 순서를 보장하기 위한 것이다.

## 컨트롤러 실패 감시

`rosbot_controller/launch/controller.launch.py`에는 컨트롤러 상태를 감시하는 로직이 있다.

```python
# 컨트롤러가 비활성화되면 전체 시스템 종료
# → 하드웨어 오류를 빠르게 감지하기 위함
```

컨트롤러가 `active` 상태에서 다른 상태로 바뀌면 launch 시스템이 종료된다.
